## 二、六宫格分镜图的整体流程

结论：六宫格分镜图不是分别生成 6 张图片后再拼接，而是先生成 6 个面板描述，再一次性生成一张完整的 `3x2` 六宫格图。

### 1. 先生成 6 个面板描述

实现入口：

- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L653)

函数：

- `generateNineGridPanels(...)`

说明：

- 虽然函数名叫 `generateNineGridPanels`，但它支持 `4 / 6 / 9` 三种网格布局。
- 当 `panelCount = 6` 时，会解析为 `3x2`，标签为“六宫格”。

布局定义位置：

- [services/ai/storyboardPromptTemplates.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/storyboardPromptTemplates.ts#L24)

### 2. 再一次性生成整张六宫格图片

实现入口：

- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L775)

函数：

- `generateNineGridImage(...)`

说明：

- 代码会把全部面板描述拼成一个总 prompt。
- 然后只调用一次 `generateImage(...)` 生成最终网格图。
- 这意味着图像模型拿到的是“一张完整 storyboard grid 图片”的要求，而不是 6 次单图请求。

关键调用：

- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L832)
- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L851)

### 3. 后续可以从整张图中裁剪单个面板

说明：

- 如果用户选择其中某一格作为首帧，系统会从整张网格图里裁剪出该 panel。
- 这进一步证明前面得到的是“一张整图”。

关键位置：

## 三、是否存在六宫格分镜图相关的 system prompt

结论分三层：

1. 有网格分镜拆分用的 `system prompt` 模板配置。
2. 没有为“六宫格”单独写一份专属 system prompt。
3. 当前运行时并没有把它作为真正的 `role: system` 发给聊天模型，而是与 user prompt 拼接成一段普通文本发送。

### 1. 存在可配置的 system 模板

默认模板定义：

- [services/ai/storyboardPromptTemplates.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/storyboardPromptTemplates.ts#L59)

内容：

```text
你是专业分镜师。请把同一镜头拆成{panelCount}个不重复视角，用于{gridLayout}网格分镜。保持同一场景与角色连续性。
```

可配置模板定义：

- [services/promptTemplateService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/promptTemplateService.ts#L143)
- [services/promptTemplateService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/promptTemplateService.ts#L331)

模板路径：

- `nineGrid.splitSystem`

### 2. 六宫格并没有独立模板

说明：

- 四宫格、六宫格、九宫格共用同一套模板。
- 区别只在运行时注入的变量不同：
  - `panelCount`
  - `gridLayout`

对于六宫格：

- `panelCount = 6`
- `gridLayout = 3x2`

变量注入位置：

- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L664)
- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L684)

### 3. 运行时没有真正作为 system role 发送

在 `generateNineGridPanels(...)` 中：

- 先渲染 `systemPrompt`
- 再渲染 `userPrompt`
- 然后拼接成 `fullPrompt`

关键代码：

- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L684)
- [services/ai/shotService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/shotService.ts#L707)

对应逻辑可概括为：

```text
fullPrompt = systemPrompt + "\n\n" + userPrompt
```

而底层 `chatCompletion(...)` 实现中，请求体只发送了一条 `role: 'user'` 消息：

- [services/ai/apiCore.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/apiCore.ts#L281)
- [services/ai/apiCore.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/ai/apiCore.ts#L295)

所以准确结论是：

- 有 system 模板概念
- 但不是 API 层面的真正 system role

## 四、六宫格分镜图的 user prompt 模板

模板来源：

- [services/promptTemplateService.ts](/Users/jueming/github_projects/BigBanana-AI-Director/services/promptTemplateService.ts#L145)

模板原文：

```text
请将以下镜头动作拆解为{panelCount}个不同的摄影视角，用于生成一张{gridLayout}网格分镜图。

【镜头动作】{actionSummary}
【原始镜头运动】{cameraMovement}
【场景信息】地点: {location}, 时间: {time}, 氛围: {atmosphere}
【角色】{characters}
【视觉风格】{visualStyle}

输出规则（只输出JSON）：
1) 顶层为 {"panels":[...]}
2) panels 必须恰好{panelCount}项，index=0-{lastIndex}，顺序为左到右、上到下
3) 每项含 shotSize、cameraAngle、description，均不能为空
4) shotSize/cameraAngle 用简短中文；description 用英文单句（10-30词），聚焦主体、动作、构图
```

当选择六宫格时，变量会被替换为：

- `{panelCount}` -> `6`
- `{gridLayout}` -> `3x2`
- `{lastIndex}` -> `5`

## 六、六宫格分镜图完整 prompt 发送形态

虽然系统里有 `splitSystem` 和 `splitUser` 两部分模板，但真正送给聊天模型的更接近下面这种形式：

```text
你是专业分镜师。请把同一镜头拆成6个不重复视角，用于3x2网格分镜。保持同一场景与角色连续性。

请将以下镜头动作拆解为6个不同的摄影视角，用于生成一张3x2网格分镜图。

【镜头动作】...
【原始镜头运动】...
【场景信息】...
【角色】...
【视觉风格】...

输出规则（只输出JSON）：
...
```

也就是说：

- 逻辑上区分了 system/user 两段
- 传输上实际合并为了单一 user message

## 八、最终结论

### 1. 菜单层面

- 项目中确实存在“导演工作台”菜单。

### 2. 六宫格生成方式

- 不是先生成 6 张图再拼。
- 是先生成 6 个 panel 文案，再一次性生成 1 张 `3x2` 六宫格图。

### 3. prompt 配置层面

- 有网格分镜拆分的 `system prompt` 模板配置。
- 没有单独给“六宫格”写专属 system prompt。
- 六宫格只是共用模板并注入 `panelCount=6`、`gridLayout=3x2`。

### 4. 请求发送层面

- 当前实现里，所谓 system prompt 没有作为真正的 `role: system` 发送。
- 它只是先被渲染成文本，再和 user prompt 拼接成一个完整字符串发给聊天模型。
